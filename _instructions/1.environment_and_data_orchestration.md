# 1. Environment and Data Orchestration

This runbook covers task 1 of the homework: create/use the isolated environments, preprocess ShareGPT-style data, launch the verifier hidden-state server, and generate offline hidden states for EAGLE-3 training.

Sources:

- Speculators tutorial: <https://docs.vllm.ai/projects/speculators/en/latest/user_guide/tutorials/train_eagle3_offline/>
- Speculators repo: <https://github.com/vllm-project/speculators>

## What You Are Generating

For offline EAGLE-3 training, you first set up the Speculators and vLLM environments, tokenize/preprocess ShareGPT-style conversations, serve the verifier model through vLLM with hidden-state extraction enabled, then send the preprocessed samples to that server and save one `hs_*.safetensors` file per sample.

The hidden-state files are training inputs for the EAGLE-3 draft head. They are much larger than raw text because they store dense activation tensors for many tokens and layers.

## Step 0: Create The Environments If Needed

If `external/speculators` or the venvs are missing, run:

```bash
bash scripts/setup_all_envs.sh
```

At minimum for this step you need:

```text
.venvs/speculators_venv
.venvs/vllm_venv
external/speculators
```

If you have a local `.env`, load it in every terminal before running commands:

```bash
set -a
source .env

```

## Suggested Homework Defaults

Use conservative single-H100 defaults first:

```bash
export MODEL_ID="Qwen/Qwen3-8B"
export MAX_SAMPLES=3000
export SEQ_LENGTH=2048
export PREPROCESSED_DIR="$PWD/data/processed/qwen3_8b_sharegpt"
export HIDDEN_STATES_DIR="$PWD/data/hidden_states/qwen3_8b_sharegpt_3k_seq2048"
export SPECULATORS_DIR="$PWD/external/speculators"
export VLLM_ENDPOINT="http://localhost:8000/v1"
```

The tutorial example uses 8192 sequence length and 5000 samples. Do not start there unless you have plenty of disk. The homework hint suggests `max-samples=3000` and sequence length `2048`.

## Step 1: Preprocess ShareGPT Data

Use `speculators_venv`.

```bash
source .venvs/speculators_venv/bin/activate

python "$SPECULATORS_DIR/scripts/prepare_data.py" \
  --model "$MODEL_ID" \
  --data sharegpt \
  --output "$PREPROCESSED_DIR" \
  --max-samples "$MAX_SAMPLES" \
  --seq-length "$SEQ_LENGTH"
```

Expected output:

```text
data/processed/qwen3_8b_sharegpt/
|-- data-*.arrow
|-- dataset_info.json
|-- state.json
`-- token_freq.pt
```

Quick check:

```bash
du -sh "$PREPROCESSED_DIR"
find "$PREPROCESSED_DIR" -maxdepth 1 -type f | sort
```

If you use your own ShareGPT-style file instead of the built-in dataset, replace `--data sharegpt` with the path to your JSONL file.

## Step 2: Launch the vLLM Hidden-State Server

Open a second terminal. Use `vllm_venv`, not `speculators_venv`.

```bash
cd /path/to/hw3_spec_dec
set -a
source .env
set +a

export MODEL_ID="Qwen/Qwen3-8B"
export SPECULATORS_DIR="$PWD/external/speculators"

source .venvs/vllm_venv/bin/activate

CUDA_VISIBLE_DEVICES=0 python "$SPECULATORS_DIR/scripts/launch_vllm.py" \
  "$MODEL_ID" \
  -- \
  --port 8000 \
  --gpu-memory-utilization 0.90
```

Wait until the server logs show application startup completed.

Health check from another terminal:

```bash
curl http://localhost:8000/v1/models
```

Notes:

- `launch_vllm.py` wraps vLLM with the hidden-state extraction settings needed by Speculators.
- For EAGLE-3, if you do not pass `--target-layer-ids`, the tutorial says the wrapper uses sensible defaults.
- On a single H100, start with one GPU. Multi-GPU data parallelism is only useful if your server has multiple GPUs.

If `launch_vllm.py` fails with `ModuleNotFoundError: speculators`, install Speculators into the vLLM venv too:

```bash
source .venvs/vllm_venv/bin/activate
python -m pip install -e external/speculators
```

Then rerun the launch command.

## Step 3: Smoke-Test Hidden-State Generation

Back in the first terminal using `speculators_venv`, generate a tiny batch first.

```bash
source .venvs/speculators_venv/bin/activate

python "$SPECULATORS_DIR/scripts/data_generation_offline.py" \
  --model "$MODEL_ID" \
  --preprocessed-data "$PREPROCESSED_DIR" \
  --endpoint "$VLLM_ENDPOINT" \
  --output "$HIDDEN_STATES_DIR" \
  --max-samples 8 \
  --concurrency 4 \
  --validate-outputs \
  --fail-on-error
```

Check output:

```bash
find "$HIDDEN_STATES_DIR" -maxdepth 1 -name 'hs_*.safetensors' | sort | head
du -sh "$HIDDEN_STATES_DIR"
```

If the smoke test works, remove the partial smoke-test output before the full run unless you intentionally want to resume from it:

```bash
rm -rf "$HIDDEN_STATES_DIR"
mkdir -p "$HIDDEN_STATES_DIR"
```

## Step 4: Run Full Hidden-State Generation

Start with concurrency `16` on one H100. If GPU utilization is low and disk writes are keeping up, try `32`. The official tutorial notes that `--concurrency` controls both concurrent vLLM requests and concurrent disk writes.

```bash
python "$SPECULATORS_DIR/scripts/data_generation_offline.py" \
  --model "$MODEL_ID" \
  --preprocessed-data "$PREPROCESSED_DIR" \
  --endpoint "$VLLM_ENDPOINT" \
  --output "$HIDDEN_STATES_DIR" \
  --max-samples "$MAX_SAMPLES" \
  --concurrency 16 \
  --validate-outputs \
  --max-consecutive-errors 16
```

For a faster rerun after the first successful validated attempt, you can omit `--validate-outputs`:

```bash
python "$SPECULATORS_DIR/scripts/data_generation_offline.py" \
  --model "$MODEL_ID" \
  --preprocessed-data "$PREPROCESSED_DIR" \
  --endpoint "$VLLM_ENDPOINT" \
  --output "$HIDDEN_STATES_DIR" \
  --max-samples "$MAX_SAMPLES" \
  --concurrency 32 \
  --max-consecutive-errors 32
```

## Step 5: Resume After Interruption

If the job stops, keep or restart the vLLM server, then rerun the same `data_generation_offline.py` command.

The tutorial says the script detects existing `hs_*.safetensors` files and skips them, so it continues from missing indices.

Useful checks:

```bash
find "$HIDDEN_STATES_DIR" -maxdepth 1 -name 'hs_*.safetensors' | wc -l
du -sh "$HIDDEN_STATES_DIR"
```

If you see errors about stale or missing temporary hidden-state files, clear the temp directory and rerun:

```bash
rm -rf /tmp/hidden_states/*
```

## Step 6: Stop the Server

After generation is complete, stop the vLLM server with `Ctrl+C` in the server terminal.

Offline EAGLE-3 training does not need the vLLM hidden-state server running after cached hidden states have been generated.

## Step 7: Continue To Training

Once hidden states are complete, continue to [2.train_eagle3_draft_head.md](2.train_eagle3_draft_head.md).

Do not delete `data/processed/` or `data/hidden_states/`; both are needed by the training runbook.

## Troubleshooting

### Server Is Not Ready

Symptoms:

- connection refused
- OpenAI client timeout
- repeated request failures

Checks:

```bash
curl http://localhost:8000/v1/models
nvidia-smi
```

Fix:

- Wait for model loading to complete.
- Confirm the vLLM server is running in `vllm_venv`.
- Confirm `--port 8000` matches `VLLM_ENDPOINT`.

### Disk Grows Too Quickly

Hidden states are dense tensors, not text. Reduce `MAX_SAMPLES` or `SEQ_LENGTH` first.

```bash
du -sh data/hidden_states/*
df -h .
```

### Sequence Length Mismatch

If validation reports hidden-state sequence lengths do not match tokenized sequence lengths:

- confirm the vLLM version is the homework version;
- rerun preprocessing and hidden-state generation with the same `MODEL_ID` and `SEQ_LENGTH`;
- remove stale hidden-state files before regenerating from scratch.

### Low Throughput During Generation

Try these in order:

1. Increase `--concurrency` from `16` to `32`.
2. Omit `--validate-outputs` after the first successful validated run.
3. Put `HF_HOME`, `data/`, and `models/` on fast local SSD storage.
4. If multiple GPUs are available, launch vLLM with data parallelism, for example:

```bash
CUDA_VISIBLE_DEVICES=0,1 python "$SPECULATORS_DIR/scripts/launch_vllm.py" \
  "$MODEL_ID" \
  -- \
  --data-parallel-size 2 \
  --port 8000 \
  --gpu-memory-utilization 0.90
```
