# 2. Train The EAGLE-3 Draft Head

This runbook covers task 2 of the homework: train an EAGLE-3 draft head for `Qwen/Qwen3-8B` using the offline hidden states generated in step 1.

Sources:

- Speculators tutorial: <https://docs.vllm.ai/projects/speculators/en/latest/user_guide/tutorials/train_eagle3_offline/>
- Speculators repo: <https://github.com/vllm-project/speculators>

## What You Are Training

The EAGLE-3 draft head proposes future tokens cheaply. During serving, vLLM verifies those proposed tokens with the full `Qwen/Qwen3-8B` verifier. Accepted draft tokens reduce the number of expensive verifier decoding steps.

For this homework, train the draft head against the original BF16 verifier behavior first. Quantize the verifier after training, then benchmark and retune speculative draft tokens.

## Step 2: Set Paths

Use the same model, sample count, and sequence length used for hidden-state generation:

```bash
export MODEL_ID="Qwen/Qwen3-8B"

export TRAIN_RUN_DIR="$PWD/output/checkpoints/eagle3_qwen3_8b_3k_seq2048"
export LOG_DIR="$PWD/output/logs/eagle3_qwen3_8b_3k_seq2048"

```

Check inputs:

```bash

find "$HIDDEN_STATES_DIR" -maxdepth 1 -name 'hs_*.safetensors' | wc -l
```

## Step 3: Preflight The Training Config

The `speculators` `v0.5.0` `train.py` CLI does not support `--dry-run`. Do not pass that flag.

Instead, check that the installed CLI exposes the required options:

```bash
python "$SPECULATORS_DIR/scripts/train.py" --help | tee "$LOG_DIR/train_help.txt"

grep -E -- "--speculator-type|--hidden-states-path|--draft-vocab-size|--save-best|--on-missing" "$LOG_DIR/train_help.txt"
```

Then verify the required input files exist:

```bash
python - <<'PY'
import os
from pathlib import Path

preprocessed = Path(os.environ["PREPROCESSED_DIR"])
hidden = Path(os.environ["HIDDEN_STATES_DIR"])

required = [
    preprocessed / "dataset_info.json",
    preprocessed / "state.json",
    preprocessed / "token_freq.pt",
]

missing = [str(path) for path in required if not path.exists()]
hs_files = sorted(hidden.glob("hs_*.safetensors"))

print("preprocessed:", preprocessed)
print("hidden states:", hidden)
print("hidden-state files:", len(hs_files))

if missing:
    raise SystemExit("Missing preprocessed files: " + ", ".join(missing))
if not hs_files:
    raise SystemExit("No hs_*.safetensors files found")
PY
```

If files are missing, do not switch to `--on-missing generate` for the homework run. Go back to step 1 and regenerate the missing hidden-state files.

## Step 4: Run Training

Start with the homework-sized run:

```bash
CUDA_VISIBLE_DEVICES=0 python "$SPECULATORS_DIR/scripts/train.py" \
  --verifier-name-or-path "$MODEL_ID" \
  --speculator-type eagle3 \
  --data-path "$PREPROCESSED_DIR" \
  --hidden-states-path "$HIDDEN_STATES_DIR" \
  --save-path "$TRAIN_RUN_DIR" \
  --log-dir "$LOG_DIR" \
  --run-name eagle3_qwen3_8b_3k_seq2048 \
  --draft-vocab-size 32000 \
  --epochs 5 \
  --lr 1e-4 \
  --total-seq-len "$SEQ_LENGTH" \
  --on-missing raise \
  --save-best \
  --seed 42 \
  2>&1 | tee "$LOG_DIR/train.log"
```

Notes:

- Keep `--on-missing raise` so training uses the offline hidden-state cache and fails loudly if data is incomplete.
- `--save-best` asks the trainer to keep a best checkpoint when supported by the installed CLI.
- The default EAGLE-3 target layers match the default `launch_vllm.py` hidden-state extraction layers. If you used custom `--target-layer-ids` in step 1, pass the exact same values here.
- The homework reference result used epoch index `4`, meaning the fifth epoch.
- If training resumes from an old checkpoint unexpectedly, add `--no-resume-from-checkpoint` for a clean run, or change `TRAIN_RUN_DIR`.

## Step 5: Monitor Validation Metrics

Watch the log:

```bash
tail -f "$LOG_DIR/train.log"
```

Search the completed log for validation metrics:

```bash
grep -E "val/(loss|full_acc|cond_acc)" "$LOG_DIR/train.log" | tail -80
```

Metrics to record for the notebook:

```text
val/loss_0_epoch
val/full_acc_0_epoch
val/cond_acc_0_epoch
val/loss_1_epoch
val/full_acc_1_epoch
val/cond_acc_1_epoch
val/loss_2_epoch
val/full_acc_2_epoch
val/cond_acc_2_epoch
val/loss_epoch
```

Interpretation:

- `full_acc_k`: the draft prediction at position `k` is correct in the full speculative chain.
- `cond_acc_k`: the draft prediction at position `k` is correct conditional on earlier speculative positions being correct.
- Later positions are harder because errors compound and the draft is predicting farther into the future from less certain context.

## Step 6: Select The Best Checkpoint

List checkpoints:

```bash
find "$TRAIN_RUN_DIR" -maxdepth 2 -type f \( -name 'config.json' -o -name '*.safetensors' -o -name '*.bin' \) | sort
find "$TRAIN_RUN_DIR" -maxdepth 2 -type d | sort
```

Choose the checkpoint with the best validation loss and reasonable position-wise accuracy. For serving, define a stable path:

```bash
export BEST_DRAFT_MODEL="$TRAIN_RUN_DIR"
```

If training writes separate epoch/checkpoint subdirectories, set `BEST_DRAFT_MODEL` to the best one instead, for example:

```bash
export BEST_DRAFT_MODEL="$TRAIN_RUN_DIR/checkpoint-epoch-4"
```

Validate the selected directory:

```bash
test -f "$BEST_DRAFT_MODEL/config.json" && echo "config ok"
find "$BEST_DRAFT_MODEL" -maxdepth 1 -type f | sort
```

Use this exact `BEST_DRAFT_MODEL` path as `DRAFT_MODEL` in [4.serve_and_benchmark_with_vllm.md](4.serve_and_benchmark_with_vllm.md).

## Step 7: If First-Position Accuracy Is Very Low

Do not tune random training hyperparameters first. Inspect the data pipeline:

1. Confirm hidden states were generated from the same `MODEL_ID` and tokenizer.
2. Confirm `SEQ_LENGTH` matches preprocessing, hidden-state generation, and training.
3. Confirm custom `--target-layer-ids`, if any, match between `launch_vllm.py` and `train.py`.
4. Confirm hidden-state files are complete and not stale.
5. Regenerate a small hidden-state smoke-test batch with `--validate-outputs`.

Then rerun training.

## Troubleshooting

### Missing Hidden States

Symptom:

```text
missing hidden state
```

Fix:

```bash
find "$HIDDEN_STATES_DIR" -maxdepth 1 -name 'hs_*.safetensors' | wc -l
```

If files are missing, rerun hidden-state generation from step 1.

### `unrecognized arguments: --dry-run`

The `v0.5.0` training script does not support `--dry-run`. Use the preflight checks in step 3, then run the step 4 training command without `--dry-run`.

### CUDA Out Of Memory

Checks:

```bash
nvidia-smi
```

Fixes:

- stop vLLM servers or quantization jobs;
- lower batch-related training settings only after checking the tutorial defaults;
- use a clean H100 session.

### Training Metrics Look Bad

If `val/full_acc_0_epoch` is far below the reference range, investigate preprocessing and hidden-state generation first. If position 0 is plausible but later positions are poor, that is expected; compare position-wise metrics instead of only total loss.
