# 4. Serve and Benchmark With vLLM

This runbook covers task 4 of the homework: serve and benchmark baseline, EAGLE-3 speculative decoding, FP8 quantization, and FP8 + speculative decoding.

Sources:

- vLLM `bench serve` docs: <https://docs.vllm.ai/en/latest/cli/bench/serve/>
- vLLM speculative decoding docs: <https://docs.vllm.ai/en/latest/features/speculative_decoding/>
- Speculators README: <https://github.com/vllm-project/speculators>

## What You Need Before Starting

You should already have:

```text
.venvs/vllm_venv
output/checkpoints/<best-draft-checkpoint>
models/Qwen3-8B-FP8-Dynamic
benchmarks/results/
```

The draft checkpoint must be a vLLM-loadable Speculators/EAGLE-3 model directory. It should contain at least a `config.json` and model weights.

If `vllm_venv` is missing:

```bash
bash scripts/setup_vllm_env.sh
```

## Step 1: Activate vLLM and Set Paths

Run from the project root:

```bash
cd /path/to/hw3_spec_dec

set -a
source .env
set +a

source .venvs/vllm_venv/bin/activate

python - <<'PY'
import vllm
print("vllm", vllm.__version__)
PY
```

If your `.env` or shell has old helper variables from data generation, remove unknown `VLLM_*` variables before starting `vllm serve`:

```bash
unset VLLM_ENDPOINT
```

vLLM warns about unknown environment variables beginning with `VLLM_`. `VLLM_ENDPOINT` is not a vLLM setting; use `SPECULATORS_ENDPOINT` in step 1 for hidden-state generation instead.

Set common variables:

```bash
export BASE_MODEL="Qwen/Qwen3-8B"
export FP8_MODEL="$PWD/models/Qwen3-8B-FP8-Dynamic"

# Change this to the actual best checkpoint selected after training.
export DRAFT_MODEL="$PWD/output/checkpoints/eagle3_qwen3_8b_3k_seq2048/4"

export HOST="127.0.0.1"
export PORT="8000"
export BENCH_DIR="$PWD/benchmarks/results"

mkdir -p "$BENCH_DIR"
```

Run this common variable block in every terminal you use for benchmarking. Shell variables are per terminal; Terminal B will not inherit `HOST`, `PORT`, `BASE_MODEL`, `FP8_MODEL`, `DRAFT_MODEL`, `SPEC_TOKENS`, or `BENCH_DIR` from Terminal A.

Verify paths:

```bash
test -d "$FP8_MODEL" && echo "FP8 model exists: $FP8_MODEL"
test -d "$DRAFT_MODEL" && echo "Draft model exists: $DRAFT_MODEL"
find "$DRAFT_MODEL" -maxdepth 1 -type f | sort
```

If your best checkpoint is named differently, update `DRAFT_MODEL`.

## Step 2: Confirm CLI Flags On The Server

vLLM flags can move between versions. Before the benchmark run, check the installed CLI:

```bash
vllm serve --help | grep -E "speculative-config|prefix-caching|served-model-name|gpu-memory"
vllm bench serve --help | grep -E "dataset-name|dataset-path|max-concurrency|save-result|result-dir|seed"
```

The commands below assume:

- `--speculative-config` is available;
- `--no-enable-prefix-caching` is available;
- `vllm bench serve` supports the HF dataset path used in the homework.

## Step 3: Shared Benchmark Command

Use the same benchmark settings for every run:

```bash
run_bench() {
  local label="$1"
  local model_name="$2"

  vllm bench serve \
    --backend openai \
    --host "$HOST" \
    --port "$PORT" \
    --model "$model_name" \
    --dataset-name hf \
    --dataset-path philschmid/mt-bench \
    --num-prompts 80 \
    --max-concurrency 8 \
    --seed 0 \
    --save-result \
    --result-dir "$BENCH_DIR" \
    --result-filename "${label}.json" \
    2>&1 | tee "$BENCH_DIR/${label}.txt"
}
```

Paste this function into the terminal where you run `vllm bench serve`. Shell functions are not shared between terminals.

Health check before each benchmark:

```bash
curl "http://$HOST:$PORT/v1/models"
```

## Step 4: Benchmark Baseline BF16 Qwen3-8B

Terminal A, start server:

```bash
source .venvs/vllm_venv/bin/activate
unset VLLM_ENDPOINT

export BASE_MODEL="Qwen/Qwen3-8B"
export HOST="127.0.0.1"
export PORT="8000"
export BENCH_DIR="$PWD/benchmarks/results"
mkdir -p "$BENCH_DIR"

CUDA_VISIBLE_DEVICES=0 vllm serve "$BASE_MODEL" \
  --host "$HOST" \
  --port "$PORT" \
  --served-model-name "$BASE_MODEL" \
  --gpu-memory-utilization 0.90 \
  --no-enable-prefix-caching \
  2>&1 | tee "$BENCH_DIR/server_baseline.log"
```

Terminal B, run benchmark:

```bash
source .venvs/vllm_venv/bin/activate
export BASE_MODEL="Qwen/Qwen3-8B"
export HOST="127.0.0.1"
export PORT="8000"
export BENCH_DIR="$PWD/benchmarks/results"

curl "http://$HOST:$PORT/v1/models"
run_bench "baseline_bf16" "$BASE_MODEL"
```

Stop the server in Terminal A with `Ctrl+C`.

The baseline is already provided in the notebook, but rerunning it on the same server is useful to confirm your environment.

## Step 5: Benchmark EAGLE-3 Speculative Decoding

Start with `num_speculative_tokens=1`, then try `2`, `3`, and possibly `4`. The homework reference used `2` for the unquantized speculative run, but you need to tune it yourself.

Terminal A:

```bash
source .venvs/vllm_venv/bin/activate
unset VLLM_ENDPOINT

export BASE_MODEL="Qwen/Qwen3-8B"
export DRAFT_MODEL="$PWD/output/checkpoints/eagle3_qwen3_8b_3k_seq2048/4"
export HOST="127.0.0.1"
export PORT="8000"
export BENCH_DIR="$PWD/benchmarks/results"
mkdir -p "$BENCH_DIR"

export SPEC_TOKENS=2
export SPEC_CONFIG='{"method":"eagle3","model":"'"$DRAFT_MODEL"'","num_speculative_tokens":'"$SPEC_TOKENS"'}'

CUDA_VISIBLE_DEVICES=0 vllm serve "$BASE_MODEL" \
  --host "$HOST" \
  --port "$PORT" \
  --served-model-name "$BASE_MODEL" \
  --gpu-memory-utilization 0.90 \
  --no-enable-prefix-caching \
  --speculative-config "$SPEC_CONFIG" \
  2>&1 | tee "$BENCH_DIR/server_spec_bf16_n${SPEC_TOKENS}.log"
```

Terminal B:

```bash
source .venvs/vllm_venv/bin/activate
export BASE_MODEL="Qwen/Qwen3-8B"
export HOST="127.0.0.1"
export PORT="8000"
export BENCH_DIR="$PWD/benchmarks/results"
export SPEC_TOKENS=2
mkdir -p "$BENCH_DIR"

curl "http://$HOST:$PORT/v1/models"
run_bench "spec_bf16_n${SPEC_TOKENS}" "$BASE_MODEL"
```

Stop the server, change `SPEC_TOKENS`, and repeat. Keep the best output-token-throughput result that also has reasonable TPOT and acceptance behavior.

Alternative if your trained Speculators checkpoint has a complete `speculator_config` that directly references the verifier:

```bash

  --host "$HOST" \
  --port "$PORT" \
  --gpu-memory-utilization 0.90 \
  --no-enable-prefix-caching
```

Use the explicit `--speculative-config` path first because it makes the verifier model and speculative token count obvious.

Also include a short explanation:

## Troubleshooting

### Server Fails To Start With Speculative Config

First print the JSON:

```bash
echo "$SPEC_CONFIG" | python -m json.tool
```

Then verify the draft model path:

```bash
test -f "$DRAFT_MODEL/config.json" && python -m json.tool "$DRAFT_MODEL/config.json" | head -80
```

If the draft model is not a complete vLLM-loadable Speculators checkpoint, go back to training/export and use the checkpoint directory that contains a complete Hugging Face-style config and safetensors weights.

### `--no-enable-prefix-caching` Is Not Recognized

Use the installed help output:

```bash
vllm serve --help | grep -i prefix
```

If the installed version uses a different flag, use that version's disabling flag. If no disabling flag exists, record that prefix caching could not be disabled in your benchmark notes.

### Benchmark Cannot Reach The Server

If curl says:

```text
curl: (3) URL rejected: No host part in the URL
```

then `HOST` or `PORT` is empty in that terminal. Re-export the common variables in Terminal B before running curl or `run_bench`.

Checks:

```bash
echo "HOST=$HOST PORT=$PORT"
curl "http://$HOST:$PORT/v1/models"
nvidia-smi
```

Fixes:

- export `HOST`, `PORT`, and `BENCH_DIR` in every terminal;
- wait until model loading is complete;
- confirm server and benchmark use the same `HOST` and `PORT`;
- check that the server did not exit after an OOM.

### FP8 + Speculative Is Slower Than FP8

Try lower draft tokens first:

```text
num_speculative_tokens=1
```

If that is still slower, inspect acceptance metrics. A poorly matched draft head or changed FP8 verifier logits can reduce accepted tokens enough that drafting overhead dominates.
