# 4. Serve and Benchmark With vLLM

This runbook covers task 4 of the homework: serve and benchmark baseline, EAGLE-3 speculative decoding, FP8 quantization, and FP8 + speculative decoding.

Sources:

- vLLM `bench serve` docs: <https://docs.vllm.ai/en/latest/cli/bench/serve/>
- vLLM speculative decoding docs: <https://docs.vllm.ai/en/latest/features/speculative_decoding/>
- Speculators README: <https://github.com/vllm-project/speculators>

You should already have:


The draft checkpoint must be a vLLM-loadable Speculators/EAGLE-3 model directory. It should contain at least a `config.json` and model weights.

## Step 2: Confirm CLI Flags On The Server

vLLM flags can move between versions. Before the benchmark run, check the installed CLI:

```bash
vllm serve --help=all | grep -E "speculative-config|prefix-caching|served-model-name|gpu-memory|max-num-seqs|max-num-batched-tokens|generation-config"
vllm bench serve --help=all | grep -E "dataset-name|dataset-path|max-concurrency|num-prompts|ignore-eos|save-result|result-dir|seed"
```

The commands below assume:

- `--speculative-config` is available;
- `--no-enable-prefix-caching` is available;
- `--generation-config vllm` is available;
- `--max-num-seqs` and `--max-num-batched-tokens` are available;
- `vllm bench serve` supports the HF dataset path used in the homework.

## Step 3: Shared Benchmark Command

Use the same benchmark settings for every measured run. The defaults below intentionally use more load than the smoke-test run: `80` prompts at concurrency `8` can under-saturate an H100 and make the FP8 thresholds look worse than they are.

```bash
run_bench() {
  local label="$1"
  local model_name="$2"

  vllm bench serve \
    --backend openai \
    --host "$HOST" \
    --port "$PORT" \
    --model "$model_name" \
    --dataset-name hf \
    --dataset-path philschmid/mt-bench \
    --num-prompts "$BENCH_NUM_PROMPTS" \
    --max-concurrency "$BENCH_MAX_CONCURRENCY" \
    --ignore-eos \
    --seed 0 \
    --save-result \
    --result-dir "$BENCH_DIR" \
    --result-filename "${label}.json" \
    2>&1 | tee "$BENCH_DIR/${label}.txt"
}
```

Paste this function into the terminal where you run `vllm bench serve`. Shell functions are not shared between terminals.

Before each measured run, do one small warmup against the already-started server. Do not use the warmup result for the notebook:

```bash
BENCH_NUM_PROMPTS=16 BENCH_MAX_CONCURRENCY=4 run_bench "warmup" "$BASE_MODEL"
```

For FP8 runs, replace `"$BASE_MODEL"` with `"$FP8_MODEL"`.

Health check before each benchmark:

```bash
curl "http://$HOST:$PORT/v1/models"
```

## Step 8.5: If Results Are Below Thresholds

Do not judge the thresholds from the first `80` prompt / concurrency `8` smoke run. The saved results in this repo were below threshold with that light load, while the server logs showed low KV-cache usage and large TTFT outliers on some runs. First tune the benchmark load and serving limits.

Use these checks before rerunning:

```bash
nvidia-smi
grep -E "Selected .*FP8|quantization=compressed-tensors" "$BENCH_DIR"/server_fp8*.log || true
grep -E "generation_config|Avg generation throughput|SpecDecoding metrics" "$BENCH_DIR"/server_*.log | tail -80
```

The FP8 logs should show a compressed-tensors FP8 kernel such as `CutlassFP8ScaledMMLinearKernel`. If not, fix model loading before tuning benchmark settings.

Rerun each configuration with a concurrency sweep. Keep the same server running for all runs in one sweep, and use the best output-token-throughput result that does not have pathological TTFT/TPOT:

```bash
run_sweep() {
  local label="$1"
  local model_name="$2"

  for concurrency in 8 16 24 32; do
    export BENCH_MAX_CONCURRENCY="$concurrency"
    export BENCH_NUM_PROMPTS="$((concurrency * 20))"
    run_bench "${label}_c${concurrency}" "$model_name"
  done
}
```

Examples:

```bash
run_sweep "baseline_bf16" "$BASE_MODEL"
run_sweep "spec_bf16_n${SPEC_TOKENS}" "$BASE_MODEL"
run_sweep "fp8" "$FP8_MODEL"
run_sweep "fp8_spec_n${SPEC_TOKENS}" "$FP8_MODEL"
```

For speculative decoding, sweep draft counts and concurrency together. The server must be restarted for each new `SPEC_TOKENS` value because `--speculative-config` is read at startup.

```bash
# Terminal A: set SPEC_TOKENS, rebuild SPEC_CONFIG, restart vllm serve.
# Terminal B: run the concurrency sweep against that server.
run_sweep "spec_bf16_n${SPEC_TOKENS}" "$BASE_MODEL"
```

For FP8 + speculative decoding, repeat the same pattern with `"$FP8_MODEL"`. In the first saved run in this repo, the best settings were:

```text
BF16 + EAGLE-3: SPEC_TOKENS=1
FP8 + EAGLE-3:  SPEC_TOKENS=2
```

When comparing candidates:

- Prefer output token throughput first.
- Reject a run if p99 TTFT is several seconds while neighboring runs are not; rerun it once because startup or compilation noise likely leaked into the measurement.
- If larger `SPEC_TOKENS` increases accepted tokens but reduces output throughput, use the smaller value. Rejected draft tokens still cost work.
- If GPU utilization is low and there are no waiting requests, increase `BENCH_MAX_CONCURRENCY`.
- If TPOT worsens sharply as concurrency rises, keep the lower concurrency even if request throughput rises.

## Troubleshooting

### Server Fails To Start With Speculative Config

First print the JSON:

```bash
echo "$SPEC_CONFIG" | python -m json.tool
```

Then verify the draft model path:

```bash
test -f "$DRAFT_MODEL/config.json" && python -m json.tool "$DRAFT_MODEL/config.json" | head -80
```

If the draft model is not a complete vLLM-loadable Speculators checkpoint, go back to training/export and use the checkpoint directory that contains a complete Hugging Face-style config and safetensors weights.

### `--no-enable-prefix-caching` Is Not Recognized

Use the installed help output:

```bash
vllm serve --help | grep -i prefix
```

If the installed version uses a different flag, use that version's disabling flag. If no disabling flag exists, record that prefix caching could not be disabled in your benchmark notes.

### Benchmark Cannot Reach The Server

If curl says:

```text
curl: (3) URL rejected: No host part in the URL
```

then `HOST` or `PORT` is empty in that terminal. Re-export the common variables in Terminal B before running curl or `run_bench`.

Checks:

```bash
echo "HOST=$HOST PORT=$PORT"
curl "http://$HOST:$PORT/v1/models"
nvidia-smi
```

Fixes:

- export `HOST`, `PORT`, and `BENCH_DIR` in every terminal;
- wait until model loading is complete;
- confirm server and benchmark use the same `HOST` and `PORT`;
- check that the server did not exit after an OOM.

### FP8 + Speculative Is Slower Than FP8

Try lower draft tokens first:

```text
num_speculative_tokens=1
```

If that is still slower, inspect acceptance metrics. A poorly matched draft head or changed FP8 verifier logits can reduce accepted tokens enough that drafting overhead dominates.
