# 3. Quantize Qwen3-8B To FP8

This runbook covers task 3 of the homework: quantize `Qwen/Qwen3-8B` with FP8 dynamic quantization using `llmcompressor`.

## What You Are Creating

You will create a separate local verifier model directory:

```text

```

Do not overwrite the original `Qwen/Qwen3-8B` checkpoint. The quantized model should keep the tokenizer and model config, add quantization metadata, and store FP8 quantized linear-layer weights. Activations are quantized dynamically at runtime.

Expected homework properties:

| Property | Expected value |
| --- | --- |
| Quantization method | compressed tensors |
| Weight format | FP8 |
| Activation format | dynamic FP8 |
| Target modules | linear layers |
| Ignored module | `lm_head` |

## Step 3: Run FP8 Dynamic Quantization

This follows the llm-compressor FP8 W8A8 dynamic example: target all `Linear` layers with `FP8_DYNAMIC`, and ignore `lm_head`.

```bash
python - <<'PY'
import os
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer
from llmcompressor import oneshot
from llmcompressor.modifiers.quantization import QuantizationModifier

model_id = os.environ["MODEL_ID"]
save_dir = os.environ["FP8_MODEL_DIR"]

print(f"Loading {model_id}")
tokenizer = AutoTokenizer.from_pretrained(model_id)
model = AutoModelForCausalLM.from_pretrained(
    model_id,
    torch_dtype=torch.bfloat16,
    device_map="auto",
)

print("Applying FP8 dynamic quantization")
recipe = QuantizationModifier(
    targets="Linear",
    scheme="FP8_DYNAMIC",
    ignore=["lm_head"],
)
oneshot(model=model, recipe=recipe)

print(f"Saving quantized model to {save_dir}")
model.save_pretrained(save_dir, safe_serialization=True)
tokenizer.save_pretrained(save_dir)
PY
```

Notes:

- This flow does not need calibration data. FP8 dynamic activation quantization computes activation scales dynamically.
- On an H100, using `device_map="auto"` should place the model on GPU. If memory is tight because another process is running, stop that process first.
- This step can take several minutes and will write a large model directory.

```bash

import json
import os
from pathlib import Path

model_dir = Path(os.environ["FP8_MODEL_DIR"])
config_path = model_dir / "config.json"
cfg = json.loads(config_path.read_text())

quant_cfg = (
    cfg.get("quantization_config")
    or cfg.get("compression_config")
    or cfg.get("quantization")
)

print("config:", config_path)
print("quantization metadata:")
print(json.dumps(quant_cfg, indent=2)[:4000])

if not quant_cfg:
    raise SystemExit("No quantization metadata found in config.json")
```

Do not treat absence of `lm_head` in grep output as proof of failure. The important checks are:

- quantization metadata exists;
- vLLM can load the model;

## Step 6: Record What To Put In The Notebook

For task 3, keep these notes for the final explanation:

- FP8 dynamic quantization is useful on H100 because Hopper GPUs have strong FP8 support and the verifier becomes less memory-bandwidth-heavy.
- `lm_head` is often excluded because it is large, numerically sensitive, and directly controls logits; preserving it can reduce quality and acceptance-rate regressions.
- Quantization can change verifier logits slightly. Speculative decoding is lossless relative to the verifier it actually runs, but changed verifier numerics can alter acceptance behavior and the best `num_speculative_tokens`.

## Troubleshooting

### CUDA Out Of Memory

Checks:

```bash
nvidia-smi
```

Fixes:

- stop vLLM servers or training jobs;
- lower other GPU memory users;
- rerun quantization on a clean H100 session.

### Hugging Face Auth Fails

If model download fails:

```bash
huggingface-cli login
```

or set this in `.env`:

```bash
export HF_TOKEN=hf_xxx
```

### vLLM Does Not Recognize The Quantized Model

Check the config:

```bash
python -m json.tool "$FP8_MODEL_DIR/config.json" | less
```

Look for quantization/compression metadata. If it is missing, rerun the quantization step and confirm `oneshot(...)` completed before saving.

### FP8 Benchmark Is Below Target

First confirm vLLM loaded the quantized model as FP8/compressed-tensors. In the task 4 benchmark logs, this should show up during server startup:

```bash
grep -E "quantization=compressed-tensors|Selected .*FP8" benchmarks/results/server_fp8*.log
```

A healthy H100 run should show compressed-tensors quantization and an FP8 kernel, for example a `CutlassFP8...` kernel. If those lines are missing, fix model loading before rerunning benchmarks.

If the FP8 kernel is selected but throughput is low, continue with the tuning pass in [4.serve_and_benchmark_with_vllm.md](4.serve_and_benchmark_with_vllm.md): use `--generation-config vllm`, increase benchmark concurrency, increase `--max-num-seqs`, and rerun the FP8 and FP8 + speculative sweeps. Do not re-quantize unless the saved `config.json` lacks quantization metadata.
